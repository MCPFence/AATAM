# AATAM - 智能体应用威胁分析模型

## 概述

AATAM是一个以实体为中心的AI智能体应用威胁建模框架。它用统一的"核心实体"模型取代了基于分层的视图，使得架构、数据流和风险可以按实体及其交互进行统一分析。该框架帮助您利用当前研究（2023-2026）识别漏洞、分析攻击路径，并生成可操作的安全检查清单。

该技能适用于：
- 智能体系统的安全审查和渗透测试
- LLM驱动的智能体和工具的威胁建模
- 跨用户、环境、扩展和生态系统的风险评估
- 攻击路径分析和缓解规划

## 快速入门

分析智能体应用时，请遵循以下工作流程：

1. 明确范围和目标（全面评估 vs 针对性评估）
2. 绘制核心实体和信任边界（Agent、Environment、Ecosystem、Expansion、User）
3. 追踪实体边界间的数据流（流入Agent、从Agent流出）
4. 对流入边界应用STRIDE模型，对流出影响应用HARMS模型
5. 使用DREAD评分评估风险并排定优先级
6. 生成以实体为中心的检查清单和攻击路径

## 核心组件

### 威胁场景框架

| 变量 | 定义 |
|------|------|
| **M** | 大语言模型驱动的智能体 |
| **A** | 攻击者的输入 |
| **B** | 智能体的输出 |
| **Impact(X, S)** | 评估输入X对系统S影响的函数 |
| **T** | 攻击者的目标 |

**攻击场景：**
- **定向攻击**：Argmax(Impact(M(A), T)) - 最大化对特定目标的影响
- **非定向攻击（智能体失控）**：Argmax(Impact(B, S)) - 智能体生成最大化系统影响的输出

### 实体关系模型

```
User ──Use──> Agent
                │
                ├─Drive─> LLM（提供智能）
                │
                └─Influence─> Environment <──> Agent（反馈循环）

Agent ──Extend Boundary──> Expansion ──Depend──> Ecosystem

Ecosystem 包含多个可协作的单智能体系统
```

**核心实体：**
1. **AI Agent（AI智能体）** - 由大语言模型驱动的自主智能体
2. **Environment（环境）** - 智能体运行的外部世界或系统
3. **Ecosystem（生态系统）** - 智能体依赖的外部生态系统
4. **Expansion（扩展）** - 用于能力、功能和知识获取的外部扩展
5. **User（用户）** - 与智能体系统交互的人类用户
（LLM能力作为AI智能体实体的一部分进行分析，不作为单独的层建模）

## 分析工作流程

### 第一步：定义目标

| 评估方法 | 优势 | 劣势 | 适用场景 |
|----------|------|------|----------|
| **全面风险评估** | 深度分析；发现最大潜在风险 | 流程复杂；耗时 | 关键系统；高风险应用；合规要求 |
| **针对性风险评估** | 成本较低；适应业务动态 | 仅关注特定风险 | 特定威胁调查；快速评估；迭代开发 |

### 第二步：数据流分析（实体边界和信任边界）

关注流入Agent的边界（来自User、Expansion、Ecosystem）和从Agent流出的边界（到Environment、Expansion、Ecosystem、User）。对于每个边界，记录：
- 跨越的信任边界
- 数据类型和模式
- 认证/授权方式
- 副作用（文件写入、网络调用、状态更新）

应用模型：
- 对流入Agent的边界应用STRIDE
- 对从Agent流出到用户/环境的影响应用HARMS

### 第三步：风险评估

使用DREAD对每个实体边界上的每个威胁进行评分。将5个因素（1-10分）相加以确定优先级：
- Damage（损害程度）、Reproducibility（可复现性）、Exploitability（可利用性）、Affected Users（受影响用户）、Discoverability（可发现性）

风险等级：
- 严重 40-50，高 30-39，中 20-29，低 10-19

## 核心实体

1) AI Agent — 由LLM驱动的自主智能体（规划、记忆、工具）
   - 架构：提示词/系统策略、规划/执行循环、记忆（短期/长期）、工具/代码使用、连接器
   - 攻击面：提示词/上下文、工具调用、文件I/O、代码执行、连接器/API、记忆状态
   - 主要威胁：提示词/间接注入、越狱/权限提升、记忆投毒、数据泄露、凭证泄露
   - 缓解措施：能力范围受限的工具、审批关卡、内容和策略守卫、输出/工具模式加固、OS/容器沙箱、最小权限、密钥隔离、审计

2) Environment — 智能体观察/操作的外部系统（文件、OS、服务、网络、设备）
   - 攻击面：文件系统、Shell/OS、HTTP/SSRF、数据库、云API、浏览器、设备/机器人
   - 主要威胁：SSRF/RCE、横向移动、工具滥用/RCE、配额耗尽、持久化/状态破坏、敏感数据暴露
   - 缓解措施：沙箱（文件系统、进程、网络）、出站白名单、默认只读、每工具凭证、速率限制/配额、防篡改日志

3) Ecosystem — 解决方案依赖的第三方平台和基础设施（模型提供商、API中心、CI/CD、包注册表）
   - 攻击面：SDK/客户端、托管端点、Webhook、包/依赖链、模型路由器
   - 主要威胁：供应链投毒、依赖混淆/拼写劫持、令牌泄露、版本降级、端点漂移/模式变更
   - 缓解措施：SBOM和版本锁定、校验和/签名验证、来源/证明、密钥轮换、提供商白名单、入站/出站模式验证

4) Expansion — 可扩展能力面（插件/技能、MCP服务器、RAG/检索器、自定义工具）
   - 攻击面：扩展安装/更新、清单、工具定义、检索器来源、提示词模板
   - 主要威胁：恶意扩展、任意文件写入/读取、通过工具逃逸沙箱、RAG间接提示词注入/数据投毒、权限过宽
   - 缓解措施：签名制品和清单审查、显式权限清单、每扩展沙箱、检索白名单/净化器、内容扫描和去重、分阶段发布

5) User — 人类操作者和最终用户
   - 攻击面：自由格式输入、上传、覆盖控制、审批、反馈
   - 主要威胁：社会工程、有害/不安全内容、同意/隐私侵犯、通过审批提升权限、提示词泄露密钥
   - 缓解措施：RBAC和职责分离、带验证的结构化输入、脱敏/PII最小化、安全输出过滤器、高风险操作审批并剥离上下文、审计跟踪

## 实体中心漏洞快速参考

### AI Agent
- 提示词/间接注入；越狱/权限提升；记忆投毒；工具滥用/RCE；密钥泄露；不安全代码执行

### Environment
- SSRF/RCE；横向移动；状态破坏和持久化；数据泄露；配额耗尽

### Ecosystem
- 依赖混淆/拼写劫持；SDK/客户端配置错误；令牌泄露；版本降级；Webhook滥用

### Expansion
- 恶意扩展；任意文件读取/写入；通过工具逃逸沙箱；RAG间接提示词注入；数据投毒；权限过宽

### User
- 社会工程；不安全输出；隐私/同意侵犯；审批滥用；通过提示词泄露密钥

- [ ] 提供商白名单和密钥轮换
- [ ] Webhook/认证加固；模式验证

#### 1.4 Expansion
- [ ] 签名扩展；清单审查和权限
- [ ] 每扩展沙箱；RAG来源白名单和净化
- [ ] 检索数据的内容扫描/脱敏

#### 1.5 User
- [ ] RBAC；高风险操作审批流程
- [ ] 结构化输入和验证；PII最小化

- 路径：Ecosystem:API -> Agent -> Environment:HTTP -> 内部服务
  - 风险：高（DREAD 36）

## 执行步骤

使用此技能时：

1. **询问澄清性问题**，了解Agent系统架构：
   - 单智能体还是多智能体系统？
   - 有哪些组件？（LLM、工具、记忆、环境）
   - 有哪些数据源/数据汇？
   - 有哪些用户交互？
   - 存在哪些核心实体，信任边界在哪里？（AI Agent、Environment、Ecosystem、Expansion、User）
   - 有哪些扩展机制？（MCP、Skills、A2A）

2. **根据业务需求设定评估范围**：
   - 关键系统进行全面评估
   - 特定威胁进行针对性评估
   - 必要时针对特定实体进行重点评估

3. **使用实体关系模型绘制所有实体**：
   - 识别所有智能体、用户、环境、扩展
   - 记录数据源和数据汇
   - 绘制信任边界

4. **为双向创建数据流图**：
   - Agent作为端点（来自外部实体的攻击向量）
   - Agent作为源（对外部实体的恶意行为）

5. **系统性地应用威胁模型**：
   - 对攻击向量使用STRIDE（Agent作为目标）
   - 对恶意行为使用HARMS（Agent作为攻击者）
   - 参考漏洞数据库了解已知问题
   - 查阅研究论文了解最新攻击向量

6. **使用DREAD或CVSS评分并排定风险优先级**：
   - 为每个识别的威胁计算DREAD分数
   - 根据风险等级排定优先级
   - 考虑研究支持的攻击成功率

7. **生成具有具体缓解措施的可操作检查清单**：
   - 参考适当的漏洞文档章节
   - 包含经研究验证有效的缓解措施
   - 提供实施指导

8. **引用漏洞数据库和研究论文的参考文献**：
   - 链接到REFERENCE_VULNERABILITIES_NEW_CN.md中的特定漏洞类别
   - 引用相关的arXiv论文及其发现
   - 如适用，包含案例研究

## 关键研究发现（2023-2026）

- 供应链/工具投毒在扩展和技能中普遍存在
- 间接提示词注入广泛影响RAG和浏览智能体
- 记忆投毒造成跨会话持久化风险
- 治理层可以大幅降低但不能消除风险；纵深防御仍然必要

## 最佳实践

### 何时使用此技能
- 部署新智能体系统之前
- 安全架构审查期间
- 评估第三方智能体工具或MCP时
- 合规评估
- 涉及智能体的安全事件之后
- 设计多智能体系统时

### 常见误区
- 仅关注单一实体/组件，忽略跨实体交互和信任边界
- 忽略通过外部数据的间接提示词注入向量
- 低估来自工具、技能和MCP的供应链风险
- 忽略多智能体交互风险
- 忽视客户端安全
- 假设沙箱无需正确配置就足够

### 与其他框架的集成
- **OWASP LLM Top 10**：将AATAM发现映射到OWASP类别
- **MITRE ATLAS**：与对手战术和技术关联
- **NIST AI RMF**：与风险管理框架对齐
- **ISO/IEC 27001**：与信息安全管理集成
