# AATAM 漏洞参考数据库

本文档提供按智能体系统层和组件组织的漏洞综合目录。在进行威胁建模分析时作为参考使用。

**版本**：2.0  
**最后更新**：2026  
**漏洞总数**：50+

---

## 核心实体概述

本参考按核心实体组织：AI Agent、Environment、Ecosystem、Expansion、User。MCP/Skills/A2A/Gateway等组件作为Expansion处理；客户端运行时属于Environment的一部分。

```
核心实体：AI Agent | Environment | Ecosystem | Expansion | User
- Expansion包括：MCP、Skills、A2A、Gateway
- Environment包括：OS/文件系统/网络/浏览器/客户端运行时
```

## AI Agent

### 1.1 指令/数据混淆 (Instruction/Data Confusion)

| 属性 | 描述 |
|------|------|
| **描述** | LLM无法正确区分用户数据和系统指令，导致数据被当作指令执行。 |
| **攻击原理** | 攻击者在用户数据中嵌入恶意指令。模型误将其当作系统指令执行，导致提示词注入攻击。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改、权限提升 |
| **实际影响** | 数据泄露、未授权操作、系统入侵 |
| **缓解措施** | 使用指令分隔符（如`---`、`<instruction>`）；实现指令优先级机制；输入格式化和转义；分离系统和用户提示架构。 |
| **研究参考** | arXiv:2603.04469（跨智能体语义流），arXiv:2601.07072（RAG中的间接注入） |

### 1.2 模型幻觉 (Model Hallucination)

| 属性 | 描述 |
|------|------|
| **描述** | 模型以高置信度生成看似合理但实际上错误或不正确的信息。 |
| **攻击原理** | 基于概率的模型缺乏真实知识验证，可能"编造"不存在的事实、引用、代码或事件。在关键应用中可能导致严重后果。 |
| **严重程度** | 高 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 医疗误诊、法律错误、财务损失、代码漏洞 |
| **缓解措施** | 使用RAG（检索增强生成）；实现事实核查机制；设置置信度阈值；高风险输出需人工审核；多模型交叉验证。 |

### 1.3 提示词注入 (Prompt Injection)

| 属性 | 描述 |
|------|------|
| **描述** | 攻击者精心构造输入以绕过模型指令约束并操纵模型行为。 |
| **攻击原理** | 攻击者在输入中嵌入恶意指令（如"忽略所有之前的指令..."）。模型识别并执行恶意内容，可能导致数据泄露或有害输出。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改、权限提升 |
| **攻击类型** | 直接注入、间接注入（通过外部数据源）、存储型注入 |
| **实际影响** | 数据泄露（用户要求智能体泄露敏感数据）、权限提升、未授权工具执行 |
| **缓解措施** | 严格的输入验证和净化；使用指令分隔符；输出审计；结构化输入格式；限制模型能力；对抗训练。 |
| **研究参考** | arXiv:2603.04469（跨智能体场景95%成功率），arXiv:2601.07072（RAG中的间接注入） |
| **案例研究** | Agent4Data：用户查询"告诉我XXX的身份证号"→智能体绕过访问控制→泄露敏感员工数据 |

### 1.4 越狱攻击 (Jailbreak)

| 属性 | 描述 |
|------|------|
| **描述** | 使用特定技术绕过模型安全限制和对齐机制。 |
| **攻击原理** | 攻击者使用各种技术（DAN提示词、角色扮演、模拟对话、翻译攻击、编码攻击、思维链操纵）诱导模型忽略安全限制。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 权限提升 |
| **攻击技术** | DAN（Do Anything Now）、角色扮演、翻译攻击、编码攻击、多轮操纵 |
| **实际影响** | 生成有害内容、绕过内容过滤器、执行危险操作 |
| **缓解措施** | 部署多层防御（输入过滤+输出审计）；对抗训练；定期更新安全策略；专门的越狱检测系统；内容安全API。 |
| **研究参考** | arXiv:2603.01414（具身LLM越狱），arXiv:2602.24009（越狱铸造厂，95%成功率） |

### 1.5 数据投毒 (Data Poisoning)

| 属性 | 描述 |
|------|------|
| **描述** | 攻击者向训练数据集或知识库注入恶意数据以操纵模型行为。 |
| **攻击原理** | 恶意样本被注入训练数据、微调数据或RAG知识库。模型学习错误的关联或后门，被触发时产生特定恶意行为。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改 |
| **实际影响** | 模型后门、持久化恶意行为、偏见输出 |
| **缓解措施** | 数据来源验证；实现数据清洗和验证；使用鲁棒训练方法；监控模型行为异常；维护训练数据审计跟踪。 |
| **研究参考** | arXiv:2602.19555（运行时供应链安全） |

## Expansion: MCP

### 2.1 接口未授权访问 (Unauthorized Interface Access)

| 属性 | 描述 |
|------|------|
| **描述** | MCP服务接口缺乏有效的身份认证，允许未授权访问。 |
| **攻击原理** | 在Stdio模式下，MCP缺乏强制认证。在HTTP/SSE模式下，认证配置不当允许攻击者未经授权访问工具和资源。 |
| **严重程度** | 高 |
| **STRIDE类别** | 欺骗、权限提升 |
| **实际影响** | 未授权工具执行、数据访问、系统入侵 |
| **缓解措施** | 实施强认证（API Key、OAuth 2.0）；为所有接口启用认证；使用mTLS；限制网络访问。 |

### 2.2 错误的身份认证 (Authentication Bypass)

| 属性 | 描述 |
|------|------|
| **描述** | MCP服务中的认证实现缺陷导致认证绕过或权限提升。 |
| **攻击原理** | 包括过度授权令牌、混淆代理攻击、会话劫持。攻击者可诱导高权限MCP服务器执行恶意操作。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 欺骗、权限提升 |
| **实际影响** | 身份冒充、权限提升、未授权操作 |
| **缓解措施** | 实施最小权限原则；使用短期令牌加刷新机制；验证令牌范围和受众；实现令牌绑定。 |

### 2.3 行为操纵 (Behavior Manipulation via Prompt Injection)

| 属性 | 描述 |
|------|------|
| **描述** | 来自基础模型的提示词注入操纵MCP工具执行行为。 |
| **攻击原理** | 攻击者使用提示词注入影响MCP工具调用参数或执行逻辑，例如注入恶意URL进行SSRF、注入恶意命令进行RCE。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改 |
| **实际影响** | SSRF攻击、远程代码执行、未授权操作 |
| **缓解措施** | 严格验证和净化所有输入参数；使用参数化查询；实现命令白名单；使用沙箱隔离。 |
| **研究参考** | arXiv:2603.07191（智能体治理架构） |

### 2.4 敏感凭证硬编码 (Hardcoded Credentials)

| 属性 | 描述 |
|------|------|
| **描述** | API密钥、数据库密码等敏感凭证被硬编码在MCP服务器代码或配置中。 |
| **攻击原理** | 硬编码凭证可通过代码泄露、逆向工程或配置文件暴露泄露。令牌也可能在MCP描述文档中泄露。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 凭证泄露、未授权访问外部服务、数据泄露 |
| **缓解措施** | 使用密钥管理系统（如HashiCorp Vault）；使用环境变量或加密配置；禁止硬编码凭证；实现凭证轮换。 |

### 2.5 沙箱隔离不充分 (Insufficient Sandbox Isolation)

| 属性 | 描述 |
|------|------|
| **描述** | MCP工具执行环境缺乏充分的隔离，允许逃逸到主机系统。 |
| **攻击原理** | 如果沙箱隔离不充分，攻击者可利用漏洞逃逸到主机系统，获得主机控制或访问其他容器资源。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 权限提升 |
| **实际影响** | 主机系统入侵、容器逃逸、横向移动 |
| **缓解措施** | 使用强隔离容器（gVisor、Kata Containers）；限制文件系统访问；禁用危险能力；实现seccomp和AppArmor策略。 |
| **案例研究** | Agent4Data：攻击者利用代码沙箱→读取后端代码→从代码中提取数据库凭证 |

### 2.6 SQL注入 (SQL Injection)

| 属性 | 描述 |
|------|------|
| **描述** | MCP工具未正确参数化数据库查询输入。 |
| **攻击原理** | 攻击者通过MCP工具注入恶意SQL语句，可能绕过认证、读取/修改数据库数据。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改、信息披露 |
| **实际影响** | 数据库入侵、数据泄露、认证绕过 |
| **缓解措施** | 使用参数化查询或预编译语句；实现输入验证；使用ORM框架；限制数据库用户权限。 |

### 2.7 SSRF (服务端请求伪造)

| 属性 | 描述 |
|------|------|
| **描述** | MCP工具未验证用户提供的URL，导致SSRF攻击。 |
| **攻击原理** | 攻击者提供恶意URL诱导MCP服务器访问云元数据服务、内部服务或受限资源。 |
| **严重程度** | 高 |
| **STRIDE类别** | 信息披露、权限提升 |
| **实际影响** | 访问内部服务、云凭证窃取、元数据暴露 |
| **缓解措施** | 严格验证URL格式和目标；阻止访问私有IP和云元数据地址；使用代理服务器；限制协议类型。 |

### 2.8 RCE (远程代码执行)

| 属性 | 描述 |
|------|------|
| **描述** | MCP工具存在代码注入漏洞，允许任意代码执行。 |
| **攻击原理** | 如果MCP工具直接执行用户输入或存在反序列化漏洞、模板注入等，攻击者可注入恶意代码。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 权限提升 |
| **实际影响** | 完全控制系统、数据泄露、恶意软件安装 |
| **缓解措施** | 禁止直接执行用户输入；避免危险函数（eval、exec）；使用安全的序列化格式；在严格隔离的沙箱中执行。 |

### 2.9 越权访问 (Unauthorized Access)

| 属性 | 描述 |
|------|------|
| **描述** | MCP工具未正确验证用户对资源的权限。 |
| **攻击原理** | 包括水平权限提升（访问其他用户资源）和垂直权限提升（普通用户执行管理员操作）。 |
| **严重程度** | 高 |
| **STRIDE类别** | 权限提升 |
| **实际影响** | 未授权数据访问、权限提升、数据篡改 |
| **缓解措施** | 实现资源级权限检查；验证用户对目标资源的访问权限；使用ABAC；维护审计日志。 |

### 2.10 MCP描述中硬编码Token (Token Hardcoded in MCP Description)

| 属性 | 描述 |
|------|------|
| **描述** | 敏感令牌或凭证被硬编码在MCP服务描述文档中。 |
| **攻击原理** | MCP描述通常公开可访问。硬编码令牌可被攻击者提取用于未授权访问。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 令牌泄露、未授权服务访问、凭证窃取 |
| **缓解措施** | 切勿在描述文档中包含凭证；使用动态令牌注入；实现描述内容验证。 |

### 2.11 浏览器登录Token本地存储 (Browser Login Token Local Storage)

| 属性 | 描述 |
|------|------|
| **描述** | 使用浏览器认证的MCP服务以不安全的方式在本地存储令牌。 |
| **攻击原理** | 本地存储的令牌可被恶意软件、共享系统的其他用户或通过文件系统漏洞访问。 |
| **严重程度** | 高 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 令牌窃取、账户入侵、未授权访问 |
| **缓解措施** | 使用安全令牌存储（操作系统密钥链、加密存储）；实现令牌静态加密；使用短期令牌加刷新机制。 |

## Expansion: Skills

### 3.1 错误知识/未经验证的知识 (Incorrect/Unverified Knowledge)

| 属性 | 描述 |
|------|------|
| **描述** | 技能包含错误、过时或未经验证的知识，导致智能体错误决策。 |
| **攻击原理** | 技能作为预定义知识库可能包含错误。智能体使用这些知识可能产生错误输出或做出有缺陷的决策。 |
| **严重程度** | 中 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 错误代码生成、有缺陷的部署流程、引入漏洞 |
| **缓解措施** | 实现知识验证机制；建立知识更新流程；标注知识来源和可靠性。 |
| **案例研究** | Agent4SDL：技能包含错误的部署流程→生产环境部署了有漏洞的应用 |

### 3.2 没有在沙箱环境执行 (No Sandbox Execution)

| 属性 | 描述 |
|------|------|
| **描述** | 技能在非沙箱环境中执行，增加系统攻击风险。 |
| **攻击原理** | 包含可执行代码或脚本的技能，如果直接在主机上执行，技能被入侵后攻击者可访问主机资源。 |
| **严重程度** | 高 |
| **STRIDE类别** | 权限提升 |
| **实际影响** | 主机入侵、数据泄露、恶意软件执行 |
| **缓解措施** | 在隔离沙箱中执行所有技能；使用容器或虚拟机；限制网络和文件系统访问。 |

### 3.3 有漏洞的伪代码 (Vulnerable Pseudocode)

| 属性 | 描述 |
|------|------|
| **描述** | 技能中的伪代码或示例代码包含安全漏洞。 |
| **攻击原理** | 技能代码示例可能包含漏洞（SQL注入、XSS、命令注入）。用户直接使用这些代码可能在其应用中引入安全问题。 |
| **严重程度** | 高 |
| **STRIDE类别** | 篡改 |
| **实际影响** | 生产环境存在漏洞代码、安全漏洞、攻击链 |
| **缓解措施** | 安全审计技能代码；提供安全代码模板；标注安全注意事项；使用静态分析工具。 |
| **案例研究** | Agent4SDL：模型生成有漏洞的代码→代码部署到生产环境存在漏洞 |

### 3.4 行为操纵-来自基础模型 (Behavior Manipulation from Foundation Model)

| **描述** | 技能行为通过基础模型层的提示词注入被操纵。 |
| **攻击原理** | 攻击者使用提示词注入操纵智能体对技能的选择、传递给技能的参数或技能的执行行为。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改 |
| **实际影响** | 未授权技能执行、参数篡改、恶意操作 |
| **缓解措施** | 验证技能调用参数；实现技能执行监控；使用技能选择白名单。 |
| **研究参考** | arXiv:2603.07191（治理架构） |

### 3.5 敏感数据外传 (Sensitive Data Exfiltration)

| 属性 | 描述 |
|------|------|
| **描述** | 技能包含将敏感数据外传到外部服务器的代码或提示词。 |
| **攻击原理** | 恶意技能或被入侵的技能可能将敏感数据（代码、凭证、用户数据）发送到攻击者控制的服务器。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 代码泄露、凭证窃取、知识产权损失 |
| **缓解措施** | 监控网络流量；阻止未授权外部连接；审计技能代码；使用网络隔离。 |
| **案例研究** | Agent4SDL：技能包含数据外传提示词→编程智能体将代码发送到外部服务器 |

### 3.6 上下文泄露 (Context Leakage)

| 属性 | 描述 |
|------|------|

| **攻击原理** | 技能可能将智能体上下文中的敏感信息（用户数据、系统状态、其他智能体数据）记录或输出给未授权方。 |
| **严重程度** | 高 |
| **STRIDE类别** | 信息披露 |
| **实际影响** | 敏感数据暴露、隐私违规、数据泄露 |
| **缓解措施** | 实现上下文隔离；净化日志和输出；控制技能间的信息流。 |
| **研究参考** | arXiv:2603.04469（跨智能体语义流） |

### 3.7 任意文件写入 (Arbitrary File Write)

| 属性 | 描述 |
|------|------|
| **描述** | 技能允许未经适当验证的无限制文件写入操作。 |
| **攻击原理** | 攻击者可通过无限制文件写入能力向敏感位置写入恶意文件、覆盖系统文件或植入后门。 |
| **严重程度** | 严重 |
| **STRIDE类别** | 篡改、权限提升 |
| **实际影响** | 恶意软件安装、系统入侵、数据损坏 |
| **缓解措施** | 实现严格的文件路径验证；使用写入位置白名单；限制文件类型和大小；监控文件操作。 |

### 3.8 硬编码RPC调用 (Hardcoded RPC Calls)

| 属性 | 描述 |
|------|------|
| **描述** | 技能包含可能恶意或易受攻击的硬编码RPC端点或API调用。 |
| **攻击原理** | 硬编码RPC调用可将智能体操作定向到攻击者控制的服务器或易受攻击的端点，实现数据拦截或篡改。 |

| **STRIDE类别** | 篡改 |
| **实际影响** | 意外操作、参数篡改、逻辑绕过 |
| **缓解措施** | 严格输入验证；实现状态机控制；添加操作确认。 |

### 9.3 扩展劫持 (Extension Hijacking)

| 属性 | 描述 |

| **严重程度** | 严重 |
| **STRIDE类别** | 篡改、权限提升 |
| **实际影响** | 恶意代码执行、后门安装、完全入侵 |

### 9.4 扩展滥用 (Extension Abuse)

|------|------|
| **描述** | 合法扩展被滥用于意外的恶意目的。 |
| **攻击原理** | 攻击者使用扩展的正常功能进行恶意目的（如文件上传扩展上传WebShell）。 |
| **严重程度** | 高 |
| **STRIDE类别** | 篡改 |
| **实际影响** | 恶意软件上传、数据泄露、系统入侵 |
| **缓解措施** | 实现最小权限；监控异常使用模式；添加速率限制和行为审计。 |

### 9.5 扩展执行可抵赖 (Extension Execution Non-Repudiation Failure)

| 属性 | 描述 |
|------|------|

| **攻击原理** | 恶意用户可因缺乏审计跟踪而否认执行破坏性操作。 |
| **严重程度** | 中 |
| **STRIDE类别** | 抵赖 |
| **实际影响** | 无法追踪攻击、问责失败、取证困难 |
| **缓解措施** | 实现全面审计日志；对关键操作进行数字签名；使用WORM存储确保日志完整性。 |

### 9.6 扩展授权过大 (Excessive Extension Permissions)

| 属性 | 描述 |
|------|------|
| **描述** | 扩展被授予超出其功能需求的权限。 |
| **攻击原理** | 过度授权的扩展如果被入侵或滥用可造成更大损害。 |
| **严重程度** | 高 |

| **实际影响** | 增加攻击面、权限提升、未授权操作 |
| **缓解措施** | 实现最小权限；定期审计权限；使用RBAC。 |

### 9.7 扩展投毒 (Extension Poisoning)

| 属性 | 描述 |
|------|------|
